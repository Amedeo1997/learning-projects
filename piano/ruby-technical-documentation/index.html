Introduction

Ruby is a dynamic, open-source programming language with a focus on simplicity and productivity. It has an elegant syntax that is natural to read and easy to write.

Created by Yukihiro "Matz" Matsumoto in 1995, Ruby combines parts of his favorite languages (Perl, Smalltalk, Eiffel, Ada, and Lisp) to form a new language that balances functional programming with imperative programming.

puts 'Hello, world!'

Ruby is used in web applications, web servers, system utilities, database work, backups, parsing, and even processing text files.

Syntax

Ruby's syntax is clean and concise, making the code highly readable. It follows the principle of least surprise (POLA) - the language behaves in such a way as to minimize confusion for experienced users.

5.times { print "We *love* Ruby -- it's out of this world!" }
  • Everything in Ruby is an object.
  • Variables do not need to be declared before use.
  • Ruby uses end to close blocks.
Variables

In Ruby, variables are used to store information that can be referenced and manipulated in a program. They can hold different types of data, such as numbers, strings, and arrays.

name = "Ruby"

There are different types of variables in Ruby, including local variables, instance variables, class variables, and global variables.

  • Local variables start with a lowercase letter or underscore.
  • Instance variables begin with @.
  • Class variables start with @@.
  • Global variables start with $.
Control Structures

Ruby offers various control structures to direct the flow of your code, including conditionals, loops, and iterators.

if temperature > 30 puts "It's a hot day!" else puts "It's not so hot." end

Commonly used control structures are if, unless, while, for, and each iterators.

Data Types

Ruby supports a variety of data types, including simple numbers, strings, and more complex types like arrays and hashes.

days = ["Monday", "Tuesday", "Wednesday", "Thursday", "Friday"]

Understanding the different data types is crucial for effective Ruby programming.

  • Numbers can be integers or floats.
  • Strings are sequences of characters.
  • Arrays are ordered collections of objects.
  • Hashes are collections of key-value pairs.
Methods and Functions

Methods in Ruby are used to bundle one or more repeatable statements into a single unit. Methods can be defined using the def keyword and are invoked by their name.

def greet(name) puts "Hello, #{name}!" end greet("Ruby")

Methods can take parameters and return values. The last expression evaluated in the method will be its return value.

  • Defining a method: def method_name; end
  • Method parameters: def method_name(param1, param2); end
  • Return values explicitly using return keyword or implicitly with the last executed expression.
Modules and Mixins

Modules in Ruby serve two primary purposes: namespacing and creating mixins. Namespacing allows you to define methods, classes, and constants in a separate space to avoid name clashes. Mixins provide a way to share functionality between classes without using inheritance.

Module as Namespace:

module MyModule class MyClass def my_method puts "Hello from MyClass within MyModule!" end end end MyModule::MyClass.new.my_method

Module as Mixin:

module Greetable def greet puts "Hello, I'm a #{self.class}!" end end class Person include Greetable end class Robot include Greetable end Person.new.greet Robot.new.greet
  • include keyword is used to mix a module's methods into a class.
  • Modules cannot be instantiated like classes, but they can be mixed into classes to add behavior.
  • Namespacing helps organize and group related classes and methods, preventing naming conflicts.
Object-Oriented Programming (OOP)

Object-Oriented Programming (OOP) is a programming paradigm based on the concept of "objects", which can contain data in the form of fields (often known as attributes or properties) and code in the form of procedures (often known as methods). Ruby is a pure object-oriented language, meaning everything in Ruby is an object, including primitive data types.

Defining a Class and Creating Objects:

class Dog def initialize(name) @name = name end def bark puts "#{@name} says Bark!" end end fido = Dog.new("Fido") fido.bark

Inheritance:

class Animal def eat puts "This animal is eating." end end class Cat < Animal def meow puts "Meow!" end end whiskers = Cat.new whiskers.eat whiskers.meow

Encapsulation:

class Person def initialize(name) @name = name end def name @name end def name=(name) @name = name end end person = Person.new("John") puts person.name person.name = "Mike" puts person.name
  • Classes encapsulate data for objects and define methods to interact with that data.
  • Inheritance allows a class to inherit behavior (methods) from another class.
  • Encapsulation hides the internal state of an object and requires all interaction to be performed through an object's methods, providing a clear interface for object interaction.
